# Git Operations ‚Äî zephyr-shgw

## Branch Naming

**Format**: `<ticket_key>_<change_name>`

| Component | Rule | Example |
|-----------|------|---------|
| `ticket_key` | Jira/issue key or task ID | `SHGW-001`, `task-005` |
| `change_name` | `lower_snake_case`, max 50 chars, descriptive verb+noun | `add_wifi_reconnect`, `fix_shadow_parse_crash` |

**Examples**:
- `SHGW-001_implement_wifi_manager`
- `SHGW-042_fix_zwave_frame_checksum`
- `task-005_add_rule_engine_unit_tests`

**If branch exists**: append sequence number ‚Äî `SHGW-001_implement_wifi_manager_2`

## Commit Message Format

```
<ticket_key>: <brief summary>

<body ‚Äî list of changes, wrapped at 72 chars>
```

**First line**: max 72 characters, imperative mood.

**Examples**:
```
SHGW-001: Add WiFi manager with DHCP and reconnect backoff

- Added src/wifi_mgr/wifi_mgr.c with connection lifecycle
- Added src/wifi_mgr/wifi_mgr.h with public API
- Updated src/main.c to call wifi_mgr_init()
- Added Kconfig entries for WiFi manager (CONFIG_SHG_WIFI_MGR)
- Added tests/unit/test_wifi_mgr/ with reconnect backoff tests
```

## Files to Include

- `src/**` ‚Äî source code (new and modified)
- `tests/**` ‚Äî test files
- `prj.conf`, `debug.conf`, `app.overlay` ‚Äî config changes
- `Kconfig` ‚Äî new Kconfig entries
- `CMakeLists.txt` ‚Äî build system changes
- `adlc_repo_orchestrator/context/**` ‚Äî healed context files (from heal_report.md)

## Files to Exclude

- `adlc_repo_orchestrator/workspace/**` ‚Äî pipeline runtime artifacts
- `build/**` ‚Äî build output
- `*.o`, `*.elf`, `*.hex`, `*.bin` ‚Äî compiled artifacts
- `.env`, `*.pem`, `*.key` ‚Äî secrets/credentials
- `keys/mcuboot-ec-p256.pem` ‚Äî dev signing key (already committed, don't modify)

## Pre-Flight Checks

1. `git status --porcelain` ‚Äî only expected files should be modified
2. No files from `adlc_repo_orchestrator/workspace/` are staged
3. Reviewer verdict is PASS (from review_report.md)
4. No `.pem`, `.key`, `.env` files in the changeset

## Safety Rules

- **NEVER** use `git push --force` or `git reset --hard`
- **NEVER** commit workspace files or build artifacts
- If unexpected files appear in `git status`, STOP and report
